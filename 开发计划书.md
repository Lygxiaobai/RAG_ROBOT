# RAG智能问答系统 - 开发计划书

## 项目概述

**项目名称**: 企业文档智能问答系统（RAG_ROBOT）
**项目类型**: B2B SaaS平台
**核心功能**: 基于RAG技术的企业内部文档智能检索与问答

**技术栈**:
- 后端: Go 1.21+ + Gin框架
- AI服务: OpenAI API (GPT-4/GPT-3.5)
- 向量数据库: Qdrant
- 前端: Vue3 + TypeScript
- 缓存: Redis
- 数据库: PostgreSQL/MySQL

**性能目标**:
- 支持500+并发请求
- 平均响应时间 < 1s
- 向量检索响应 < 800ms
- 问答准确率 ≥ 85%
- 系统可用性 ≥ 99.5%

## 关键技术点

### 1. 文档分块策略
- **固定长度分块**: 每500-1000字符一块，重叠100字符
- **语义分块**: 按段落、章节自然分割
- **混合策略**: 结合固定长度和语义边界

### 2. 向量检索优化
- 使用HNSW索引，ef_construct=200, m=16
- Top-K设置为5-10个最相关片段
- 相似度阈值过滤（>0.7）

### 3. Prompt工程
```
系统角色：你是一个企业文档助手，基于提供的文档内容回答问题。

上下文：{检索到的文档片段}

用户问题：{用户输入}

要求：
1. 仅基于提供的文档内容回答
2. 如果文档中没有相关信息，明确告知用户
3. 回答要准确、简洁、专业
```

### 4. 缓存策略
- **L1缓存**: 本地内存缓存（热点数据，TTL=5分钟）
- **L2缓存**: Redis缓存（常用数据，TTL=1小时）
- **缓存Key设计**: `qa:{question_hash}`, `doc:{doc_id}`, `vector:{text_hash}`

### 5. 降级策略
- OpenAI超时 → 切换到备用模型或返回检索结果
- Qdrant不可用 → 使用全文检索降级
- 缓存失效 → 直接查询数据库

## 开发规范

### 代码规范
- 遵循Go官方代码规范
- 使用golangci-lint进行代码检查
- 函数注释必须完整
- 错误处理不能忽略

### Git规范
- 分支策略: main(生产) / develop(开发) / feature/* / hotfix/*
- Commit格式: `type(scope): description`
  - feat: 新功能
  - fix: 修复bug
  - docs: 文档更新
  - refactor: 重构
  - test: 测试

### 测试规范
- 单元测试覆盖率 > 80%
- 关键路径必须有集成测试
- 性能测试必须达标
